1. Jobs API - Timeout Issue

Problem: Request times out after 30 seconds Root Cause: API returns 1 year of jobs by default (too much data)

Solution: Add completedJobLookupTime parameter

# Current (FAILS):
response = requests.get(f"{base_url}/Job", headers=headers, timeout=30)

# Fixed (WORKS):
response = requests.get(
    f"{base_url}/Job?completedJobLookupTime=86400",  # Last 24 hours
    headers=headers,
    timeout=30
)

Parameter Details: - completedJobLookupTime - Time in seconds to look back - Example values: - 86400 = Last 24 hours - 604800 = Last 7 days - 2592000 = Last 30 days

Additional Filters Available: - ?jobFilter=Backup - Filter by job type - ?statusList=Completed,Failed - Filter by status - ?clientId=123 - Filter by specific client
